# Reglas de Programación / Programming Rules

Este archivo contiene las reglas estrictas de desarrollo que la Inteligencia Artificial (IA) debe leer, comprender y seguir **SIEMPRE** antes de comenzar a escribir o modificar cualquier código en este espacio de trabajo.

---

## 📋 Reglas Obligatorias

### 1. 📏 Límite de Tamaño de Scripts
* **Regla:** Ningún archivo de script o código debe superar las **200 líneas**.
* **Propósito:** Mantener el código modular, enfocado, fácil de leer, mantener y depurar. Si un script crece demasiado, debe dividirse en módulos o componentes más pequeños.

### 2. 💬 Comentarios Línea por Línea
* **Regla:** **Cada línea de código que se programe debe estar comentada.**
* **Propósito:** Explicar el propósito exacto y el funcionamiento de cada instrucción de manera clara y explícita, facilitando el aprendizaje y la revisión del código.

### 3. ✍️ Encabezado Informativo del Script
* **Regla:** Al principio de cada script, se debe incluir obligatoriamente un **comentario de encabezado** (bloque de comentarios) que describa detalladamente:
  * Qué hace el script.
  * Su funcionalidad principal.
  * Su propósito dentro del proyecto.

---

## 🤖 Instrucción para la IA (System Prompts integration)
> [!IMPORTANT]
> **Antes de escribir cualquier línea de código o script:**
> 1. Lee este archivo (`rules.md`).
> 2. Asegúrate de cumplir estrictamente con las 3 reglas descritas arriba.
> 3. Si tu respuesta genera un script de más de 200 líneas, detente y reestructura la solución en partes más pequeñas.
